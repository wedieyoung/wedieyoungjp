# wedieyoung.jp 新サイト移行手順（ロリポップ）

旧WordPressサイトを、このリポジトリの新サイト（静的サイト）に置き換える手順です。
ドメイン・サーバー契約・画像・メールはすべて現状のまま使い続けます。

> アップロード用ファイル一式: デスクトップの `wedieyoung-site-upload.zip`
> （中身: `index.html` / `css/` / `js/` / `assets/` — 解凍してから使います）

---

## STEP 1. バックアップ（必須・15分）

1. ロリポップの「ユーザー専用ページ」にログイン
2. メニューから **「ロリポップ!FTP」** を開く（ブラウザ上のファイル管理画面）
3. wedieyoung.jp の公開フォルダを開き、**`wp-content` フォルダを丸ごとPCにダウンロード**
   - フォルダごとダウンロードできない場合は、FTPソフト（FileZilla等）を使うか、
     最低限 `wp-content/uploads`（画像置き場）と `wp-content/themes` を確保する
4. WordPress管理画面（`wedieyoung.jp/wp-admin`）→「ツール」→「エクスポート」→「すべてのコンテンツ」で記事データをダウンロード

※ このバックアップは「万一の保険」です。移行後も旧データはサーバー上に残るので、慌てて使うことはありません。

## STEP 2. 新サイトのファイルをアップロード（20分）

デスクトップの `wedieyoung-site-upload.zip` を**先に解凍**しておく。

ロリポップ!FTPで、wedieyoung.jp の公開フォルダ（`wp-admin` や `wp-content` が見える場所）に以下をアップロード:

| アップロードするもの | 置き場所 |
|---|---|
| `index.html` | 公開フォルダ直下 |
| `css` フォルダの中身（style.css） | 新規作成した `css` フォルダの中 |
| `js` フォルダの中身（data.js, main.js） | 新規作成した `js` フォルダの中 |
| `assets` フォルダの中身（images一式） | 新規作成した `assets/images` フォルダの中 |

- ロリポップ!FTPはフォルダごとのアップロードができないため、「フォルダを新規作成 → その中にファイルをアップロード」を繰り返す
- ファイル数が多くて大変な場合は、無料のFTPソフト **FileZilla** を使うとフォルダごとドラッグ&ドロップできる
  （接続情報は ユーザー専用ページ →「アカウント情報」→ FTPSサーバー・FTPアカウント・FTPパスワード）

⚠️ **`wp-content` フォルダは絶対に削除・移動・リネームしない。**
新サイトの画像の大部分は `wedieyoung.jp/wp-content/uploads/...` を直接参照しています。

## STEP 3. WordPressの入口を無効化（1分）

公開フォルダ直下にある **`index.php` を `index_old.php` にリネーム**する。

これだけで、サイトにアクセスした人には新サイト（index.html）が表示されるようになります。

## STEP 4. 動作確認（5分）

スマホとPCの両方で `https://wedieyoung.jp` を開いて確認:

- [ ] 新サイトのトップ（ロゴアニメーション）が表示される
- [ ] RELEASES の各ジャケット画像が表示される（旧サーバー画像の再利用確認）
- [ ] NEWS の記事が開ける
- [ ] EVENTS のギャラリー写真が表示される
- [ ] ブラウザの再読み込み（Ctrl+Shift+R / スマホはキャッシュクリア）をしても正常

## もし問題が起きたら（復旧方法）

`index_old.php` を `index.php` に戻すだけで、即座に旧WordPressサイトに戻ります。

## 移行後の運用

- **移行完了後、GitHub Actionsによる自動デプロイを設定する（決定済み）**: GitHubにpushすると自動でロリポップにFTPアップロードされる仕組み。設定はClaude Codeに依頼（FTPパスワードはGitHub Secretsに保存し、チャットには貼らない）
- 自動化設定までのつなぎ: `js/data.js` を編集 → ロリポップ!FTPで上書きアップロード
- **新しい画像**: `assets/images/` に追加し、data.js からファイル名で参照する

## 数週間後の掃除（任意・ただし推奨）

新サイトが問題なく動いていることを確認したら、放置されたWordPressはセキュリティリスクになるため削除する:

1. STEP 1 のバックアップが手元にあることを再確認
2. 公開フォルダから以下を削除:
   - `wp-admin` フォルダ、`wp-includes` フォルダ
   - `index_old.php` および `wp-` で始まる残りのファイル（wp-login.php, wp-config.php など）
3. **`wp-content` だけは残す**（画像が入っているため）
4. あわせてロリポップのユーザー専用ページで WordPress簡単インストールの管理対象からも外れていることを確認

※ さらに先の話として、`wp-content/uploads` の画像をこのリポジトリに取り込み、GitHub Pages へ完全移行することも可能（必要になったらClaude Codeに相談）。
